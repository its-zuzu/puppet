# CTF Login Rate Limits
# -----------------------
# Optimized for ~400 concurrent users sharing IPs (NAT/WiFi)
# Strategy: IP-based limiting to prevent credential stuffing

# Window duration (e.g., 15m, 1h)
# 15 minutes is a standard balance
LOGIN_RATE_WINDOW=15m

# Max attempts per window per IP
# 500 allows multiple users on shared WiFi to login without issues
# If using strict 1-user-per-IP, this could be lower (e.g., 5-10)
# But for CTF events, ALWAYS assume shared IPs (fail-safe)
LOGIN_RATE_MAX=500

# Note:
# - These limits are backed by Redis.
# - The system is configured to "Fail Open".
# - If Redis goes down, rate limiting is temporarily disabled
#   to ensure legitimate users can still login.
